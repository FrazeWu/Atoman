# 任务计划：Podcast 模块规划

## 目标
作为 Studio 的播客子模块，支持创作者通过频道发布播客节目（Show + Episode），复用底部播放条收听，输出符合播客规范的 RSS feed。

## 当前阶段
阶段 1：产品细节确认完成，待进入实现规划。

## 各阶段

### 阶段 1：产品细节确认
- [x] 确认 Podcast 复用 Blog 频道体系，`Channel.type = podcast` 的频道即为播客节目（Show）
- [x] 确认频道没有类型限制，任何频道都可发三种内容，type 只是内容属性
- [x] 确认 Show = 特殊频道，Episode = 该频道下发布的播客内容
- [x] 确认音频文件必须上传（本地），无外链选项
- [x] 确认 Shownotes 只保留 URL 和加粗，不需要完整 Markdown 编辑器
- [x] 确认不需要时间章节（Chapters）
- [x] 确认有草稿/发布两个状态
- [x] 确认支持定时发布
- [x] 确认可见范围：公开 / 仅关注者 / 私密
- [x] 确认频道主页分 Tab 展示：全部 / 文章 / 视频 / 播客
- [x] 确认按内容类型有独立 RSS：`/channel/:slug/rss/podcast`（含 `<enclosure>` 音频标签，符合播客规范）
- [x] 确认收听复用底部播放条，单集加入队列
- [x] 确认评论支持，复用论坛纯文本 + 图片格式
- **状态：** complete

## 已做决策
| 决策 | 理由 |
|---|---|
| Podcast 复用 Studio 频道体系 | 创作者身份统一，不重复建设 |
| Show = Channel，Episode = 频道内播客内容 | 模型简单，复用现有 Channel-Collection-Post 架构 |
| Shownotes 只保留 URL 和加粗 | 播客说明不需要复杂排版 |
| 不做时间章节 | 增加发布复杂度，首期不必要 |
| 收听复用底部播放条 | 和音乐模块共用播放体验，统一交互 |
| 独立播客 RSS 含 enclosure 标签 | 符合播客规范，可被外部客户端（Overcast 等）订阅 |
| 评论复用论坛格式 | 统一评论体验，不重复建设 |
