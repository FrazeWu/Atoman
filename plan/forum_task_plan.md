## 已做决策
| 决策 | 理由 |
|---|---|
| 社区模块采用"分区 + 主题帖 + 嵌套回复"结构 | 符合长期讨论与内容沉淀需要 |
| 分区与标签并存 | 同时支持硬分类和软聚合 |
| 点赞、收藏、订阅是基础互动能力 | 有助于提升留存和内容回流 |
| 关闭帖与置顶帖由管理能力控制 | 保持秩序与引导讨论方向 |
| 精华内容需要单独标识 | 帮助沉淀优质讨论成果 |
| 不允许匿名发帖 | 保持社区可溯源，降低治理难度 |
| 不做声望系统 | 后续单独开发用户评分系统 |
| 发帖用 Tiptap WYSIWYG，回复纯文本 + 图片 | 帖子需要排版，回复保持轻量 |
| 任何帖子可标记最佳回答，无显式帖子类型 | 不强制分类，保持发帖灵活性 |
| 最佳回答置顶显示，选后不关闭帖子 | 方便快速找到答案，同时保持讨论继续 |
| 分区一层结构 | 结构简单，避免分区过深导致内容分散 |
| 用户可申请新分区，管理员审核 | 社区驱动分区生长，保持可控 |
| 标签自由填写 | 灵活聚合，降低发帖门槛 |
| 排序：最新 / 最热 / 精华 | 覆盖发现、活跃、质量三种需求 |
| 举报：普通通知管理员，阈值后自动折叠 | 平衡效率与误伤风险 |
| 帖子随时可编辑，显示「已编辑」标记 | 保持透明，不限制修正 |
| 用户可直接删除自己的帖子 | 尊重用户内容所有权 |
| 不做分区订阅 | 降低复杂度，通知靠 @username 和回复触发 |
