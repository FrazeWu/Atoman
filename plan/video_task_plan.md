# 任务计划：Video 模块规划

## 目标
作为 Studio 的视频子模块，支持创作者通过频道发布视频内容（本地上传或外部链接），提供独立的发现页、播放页和管理页。

## 当前阶段
阶段 1：产品细节确认完成，待进入实现规划。

## 各阶段

### 阶段 1：产品细节确认
- [x] 确认 Video 为独立模块，有自己的发现页、播放页、管理页
- [x] 确认视频归属 Studio 频道体系（Channel），视频是频道发布的内容类型之一
- [x] 确认视频支持本地上传 + 外部链接（YouTube/Bilibili 等）两种方式
- [x] 确认字幕文件可选上传，非必须
- [x] 确认简介只保留 URL 和加粗，不需要完整 Markdown 编辑器
- [x] 确认有草稿/发布两个状态
- [x] 确认支持定时发布
- [x] 确认可见范围：公开 / 仅关注者 / 私密
- [x] 确认频道主页分 Tab 展示：全部 / 文章 / 视频 / 播客
- [x] 确认按内容类型有独立 RSS：`/channel/:slug/rss/video`
- [x] 确认播放页布局：YouTube 风格（播放器 + 标题，右侧推荐视频列表，下方评论）
- [x] 确认评论支持，复用论坛纯文本 + 图片格式
- **状态：** complete

## 已做决策
| 决策 | 理由 |
|---|---|
| Video 为独立模块 | 有独立的发现页、播放页、管理需求，与文章展示逻辑差异大 |
| 归属 Studio 频道体系 | 频道是通用创作身份，不限内容类型 |
| 支持本地上传 + 外部链接 | 降低存储成本，兼容已有平台内容 |
| 简介只保留 URL 和加粗 | 视频简介不需要复杂排版 |
| 播放页 YouTube 风格 | 右侧推荐视频有助于内容发现和留存 |
| 评论复用论坛格式 | 统一评论体验，不重复建设 |
| 独立 RSS 按内容类型输出 | 订阅者可以只订阅视频内容 |
