# Forum Spec

## Purpose
社区模块负责承载自由、日常、长期的讨论交流，与辩论模块形成分工：社区偏开放交流与沉淀，辩论偏结构化论证。它需要同时支持活跃发帖、长期回看、治理机制和优质内容回流。

## Core Model

### Information architecture
- 社区采用“分区 + 主题帖 + 嵌套回复”结构。
- 分区承担硬分类职责。
- 标签承担软聚合作用。
- 帖子是主要内容单元。
- 回复支持嵌套与引用回复。

### Core objects
- `ForumCategory`
- `ForumTopic`
- `ForumReply`
- `Reaction`
- `Bookmark`
- `Subscription`
- `ModerationAction`

### Governance model
- 管理能力包括置顶、关闭、隐藏、软删除、精选与用户禁言。
- 举报达到阈值后允许自动折叠。
- 社区需要反刷屏与频率限制。
- 用户可申请新分区，由管理员审核。

## Product Rules
- 不允许匿名发帖。
- 不做声望系统，用户评分未来单独实现。
- 发帖使用 Tiptap WYSIWYG 编辑器。
- 回复保持轻量，只支持纯文本 + 插入图片。
- 任何帖子都可标记最佳回答，无需单独帖子类型。
- 选出最佳回答后不关闭帖子，讨论可以继续。
- 图片支持本地上传与外链两种方式。
- 保留引用回复，支持 `@username` 触发通知。
- 分区只有一层结构，不做子分区。
- 标签自由填写。
- 排序至少包含最新 / 最热 / 精华。
- 用户可直接删除自己的帖子。
- 不做分区订阅。

## Page Structure

### Forum home
- 分区导航
- 热门帖子
- 最新帖子
- 精华帖子
- 分区申请入口

### Category page
- 分区简介
- 排序切换
- 帖子列表
- 发帖入口

### Topic detail page
- 标题、作者、时间、标签
- 正文内容
- 收藏 / 点赞 / 订阅
- 回复区（支持嵌套）
- 最佳回答置顶展示
- 管理操作入口

### Posting flow
- 标题
- 正文编辑器
- 分区选择
- 标签
- 附件/图片（可选）

## Durable Decisions
- 社区模块采用嵌套式长期讨论结构，而不是扁平评论流。
- 最佳回答是轻问答增强，不应把帖子强制分型为“问题帖”。
- 社区治理强调可持续运营，因此举报、精选、关闭、置顶都是基础能力。
- 回复编辑体验刻意保持轻量，避免正文编辑器复杂度扩散到所有互动层。

## Boundaries
- 社区不是辩论模块，不要求严格的立场树与证据化结构。
- 不做匿名模式。
- 不做分区订阅。
- 不在首期引入声望/等级系统。
