# Video Spec

## Purpose
Video 是 Studio 的视频子模块，用于让创作者通过频道发布视频内容，并提供独立的发现页、播放页与管理页。它应复用频道创作身份体系，但保留与文章不同的视频消费体验。

## Core Model

### Identity and ownership
- Video 归属 Studio 频道体系。
- 视频是频道发布的内容类型之一。
- 频道可以混合发布文章、视频、播客。

### Content rules
- 支持本地上传与外部链接（如 YouTube / Bilibili）两种方式。
- 字幕文件可选上传。
- 简介仅保留 URL 与加粗，不需要完整 Markdown 编辑器。
- 支持草稿 / 发布状态。
- 支持定时发布。
- 可见范围支持公开 / 仅关注者 / 私密。

## Discovery and playback
- Video 应有独立发现页。
- 播放页采用 YouTube 风格：播放器 + 标题，右侧推荐视频列表，下方评论。
- 频道主页按内容类型分 Tab 展示：全部 / 文章 / 视频 / 播客。
- 独立视频 RSS 为 `/channel/:slug/rss/video`。
- 评论复用论坛纯文本 + 图片格式。

## Durable Decisions
- Video 作为独立模块存在，因为发现页、播放页与管理需求与文章差异明显。
- 频道是统一创作身份，视频不应再单独引入身份体系。
- 本地上传与外部链接并存，用于平衡体验与存储成本。
- 播放页的推荐区是基础发现能力，而不是后续附加优化。
- 评论体验与论坛保持统一，避免重复建设一套评论系统。

## Open Boundaries
- 推荐视频算法仍可后续细化。
- 播放统计（播放量、完播率）仍可后续扩展。
- 是否支持视频合集/播放列表仍可增强。
