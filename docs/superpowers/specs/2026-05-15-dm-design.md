# 私信（Direct Message）系统设计规格

**日期：** 2026-05-15  
**状态：** 已审阅确认  
**模块：** DM（一对一私信）  
**依赖：** 通知系统（`2026-05-15-notification-design.md`）中规划的 `/ws/user` WebSocket 端点与 inboxStore

---

## 一、背景与范围

用户之间一对一私信能力，类似 V2EX / LinuxDo 私信功能。私信 Tab 集成在统一收件箱 `/inbox` 中。

**不在范围内：**
- 群聊 —— 不做
- 消息已读回执（"对方已读"提示）—— 二期
- 消息撤回 —— 二期
- 屏蔽用户 —— 二期
- 消息搜索 —— 二期

---

## 二、用户界面

### 入口

- **导航栏「收件箱 N」**：点击进入 `/inbox`，切换到「私信」Tab
- **用户主页**：「发私信」文字按钮，跳转至 `/inbox?tab=dm&user=:username`，自动打开与该用户的对话

### 收件箱私信 Tab（`/inbox` 第 4 个 Tab）

左右分栏布局（与通知 Tab 共用框架）：

**左侧**：会话列表，按 `last_message_at` 降序排列
- 每行：对方用户名 + 最新消息预览（截断到 50 字符）+ 时间 + 未读数角标
- 未读会话加粗显示

**右侧**：消息流
- 消息按时间升序排列（最新在底部）
- 自己的消息靠右，对方消息靠左
- 消息气泡：纯文本 或 图片（点击图片全屏预览）
- 底部：文本输入框 + 「发送」按钮 + 「上传图片」文字按钮
- 进入对话时自动标记未读消息为已读

## 三、数据模型

```go
// DMConversation 两人之间唯一的会话
type DMConversation struct {
    Base
    ParticipantA      uuid.UUID  `gorm:"type:uuid;not null;index"` // UUID 值较小的一方
    ParticipantB      uuid.UUID  `gorm:"type:uuid;not null;index"` // UUID 值较大的一方
    LastMessageAt     *time.Time
    LastMessagePreview string    `gorm:"size:100"` // 截断 100 字符，列表展示用
}
// UNIQUE (participant_a, participant_b)
// participant_a/b 按 UUID 字节值排序存储，确保两人之间只有一条记录

func (DMConversation) TableName() string { return "dm_conversations" }

// DMMessage 会话内的单条消息
type DMMessage struct {
    Base
    ConversationID uuid.UUID `gorm:"type:uuid;not null;index"`
    SenderID       uuid.UUID `gorm:"type:uuid;not null;index"`
    Sender         *User     `gorm:"foreignKey:SenderID;references:UUID"`
    Content        string    `gorm:"type:text"`  // 纯文本，纯图片消息时可为空
    ImageURL       string    `gorm:"column:image_url"` // 可选，S3/MinIO URL
    ReadAt         *time.Time
}
// Content 与 ImageURL 至少一个非空（后端校验）

func (DMMessage) TableName() string { return "dm_messages" }
```

**索引：**
- `(conversation_id, created_at)` —— 消息列表分页
- `(conversation_id, sender_id, read_at)` —— 未读数统计

### UserSettings 新增字段

```go
DMPermission string `gorm:"default:'anyone'"` // "anyone" | "following_only" | "one_before_reply"
```

## 四、发送权限控制

发送消息前后端按以下顺序检查：

| dm_permission | 检查逻辑 |
|---|---|
| `anyone` | 直接允许 |
| `following_only` | sender 必须在 recipient 的 following 列表中（查 `follows` 表）|
| `one_before_reply` | 若 sender 已在该 conversation 发过消息，且 recipient 尚未回复过 → 拒绝（`403`，错误码 `dm_waiting_reply`）|

失败响应示例：
```json
{ "error": "dm_waiting_reply", "message": "对方设置了回复前仅可发送一条消息" }
{ "error": "dm_permission_denied", "message": "仅对方关注的用户可发送私信" }
```

自发屏蔽：`sender_id == recipient_id` 时拒绝（`400`）。

## 五、实时推送（WebSocket）

复用通知模块中的 `/ws/user` 端点（inboxStore 管理连接生命周期）。

新私信推送格式：

```json
{
  "event": "dm",
  "data": {
    "conversation_id": "uuid",
    "message_id": "uuid",
    "sender_id": "uuid",
    "sender_username": "alice",
    "content": "你好...",
    "image_url": "",
    "created_at": "2026-05-15T00:10:00Z"
  }
}
```

前端 dmStore 收到 `dm` 事件后：
- `unreadCount++`（若当前非该对话）
- 若当前对话匹配 `conversation_id`：append 消息，自动标记已读

---

## 六、图片上传

复用现有 S3/MinIO storage 模块。

上传流程：
1. `POST /api/dm/upload`（multipart/form-data）→ 返回 `{ image_url: "https://..." }`
2. 前端拿到 URL 后随消息正文一并提交（或单独发一条纯图片消息）

图片大小限制：≤ 10MB（与现有博客上传限制一致）。

---

## 七、API 端点

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/dm/conversations` | 会话列表，按 `last_message_at` 降序 |
| GET | `/api/dm/conversations/:username` | 与某用户的消息列表（`?page=1`，按时间升序）|
| POST | `/api/dm/conversations/:username` | 发送消息（含权限检查，body: `{content, image_url}`）|
| POST | `/api/dm/upload` | 上传图片，返回 `{image_url}` |
| PUT | `/api/dm/conversations/:username/read` | 标记该会话所有消息为已读 |
| GET | `/api/dm/unread-count` | 私信未读数（WS 断线回退）|
| WS | `/ws/user` | 复用，event: `"dm"` |

所有端点需 JWT 鉴权（`AuthRequired` middleware）。

## 八、前端架构

### dmStore (Pinia)

```typescript
interface DMState {
  unreadCount: number
  conversations: DMConversation[]
  activeConversation: string | null  // 当前对话的 username
  messages: DMMessage[]
  loading: boolean
}
```

职责：
- `fetchConversations()` —— 加载会话列表
- `openConversation(username)` —— 加载消息历史，标记已读，设置 activeConversation
- `sendMessage(username, content, imageUrl?)` —— 发送消息
- WS `dm` 事件处理：unreadCount++，若 activeConversation 匹配则 append 并标记已读

### inboxStore（协调层，与通知模块共用）

```typescript
totalUnread = notificationStore.unreadCount + dmStore.unreadCount
```

inboxStore 统一管理 `/ws/user` 连接，按 `event` 字段分发到 notificationStore / dmStore。

### 路由与导航

```
/inbox                          InboxPage.vue（4 Tab 收件箱）
/inbox?tab=dm&user=:username    直接打开与某用户的对话
```

用户主页「发私信」按钮：
```typescript
router.push({ path: '/inbox', query: { tab: 'dm', user: username } })
```

## 九、UserSettings 隐私设置页

在用户设置页面新增「私信权限」单选：
- 任意人可私信（默认）
- 仅我关注的人
- 陌生人仅可发一条（回复前）

`PUT /api/user/settings` 接口更新 `dm_permission` 字段（UserSettings 表现有接口扩展）。
